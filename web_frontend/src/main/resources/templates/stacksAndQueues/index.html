Stacks and queues

The sub-topics covered are:

  • Custom implementation of a stack using an inner class and using an array
  • Custom implementation of a queue
  • Implementing three stacks with a single array
  • Implementation of a stack with an additional minimum() method
  • Set of stacks. Each stack has a maximum capacity and when a stack is full, another vacant stack is populated. Includes a popAt() method which returns performs pop() on a specific stack.
  • Implementing a queue using two stacks
  • Building a stack which stores the smallest element at the top of the stack
  • Animal shelter. Implementing two queues such that dequeue() returns an animal based on shelter time or type (dog or cat) only.
