Stacks and queues

The sub-topics covered are:

  • Custom implementation of a stack via a linked list (nested class), ListStack, and via an array, ArrayStack
  • Custom implementation of a queue, via a linked list
  • Implementing an array of three stacks. Each stack has a maximum capacity and when a stack is full, another vacant stack is populated.
  • Implementation of an array based stack, with an additional getMinimum() method to return the smallest int
  • Set of stacks. Each stack has a maximum capacity and when a stack is full, another vacant stack is populated. Includes a popAt() and peekAt() methods which parses a specific stack.
  • Implementing a queue using two stacks
  • Building a stack which stores the smallest element at the top of the stack
  • Animal shelter. Implementing two queues such that dequeue() returns an animal based on shelter time or type (dog or cat) only.
